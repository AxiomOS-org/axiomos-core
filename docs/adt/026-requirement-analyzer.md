# 026 Requirement Analyzer

## Purpose
Translate business requirements into structured engineering intents before generation.

## Input
- Natural language business requirement
- Domain context pack
- Constraints (timeline, risk, compliance)

## Output
- Requirement decomposition
- Acceptance criteria
- Ambiguity list
- Risk flags
- Candidate module boundaries

## Validation
- Must map every requirement to a testable outcome.
- Must identify missing constraints explicitly.
