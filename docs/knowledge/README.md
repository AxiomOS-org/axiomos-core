# Knowledge Base

## Purpose
Domain knowledge packs consumed by AI Orchestrator and ADT planners before code generation.

## Structure
```
docs/knowledge/
  Accounting/
  Inventory/
  CRM/
  HR/
  Manufacturing/
  POS/
  AI/
  Workflow/
  Automation/
  Security/
```

## Usage Rule
Before generating a domain module (e.g. Accounting), agents must load relevant knowledge pack rules and constraints.

## Content Types
- Domain glossary
- Business rules
- Regulatory/compliance notes
- Integration patterns
- Anti-patterns

## Governance
Knowledge packs are versioned and reviewed like code.
